Dashboard-first Laravel business toolkit

Install once. Open /bisbond. Configure from one place.

Laravel Bisbond is a regional business toolkit for Laravel applications. It centralizes settings, Bangla formatting, invoice preview, SMS configuration, health checks, routes, commands, and future payment integrations in one dashboard.

Get Started Developer Guide

Quick Start

Install the package and open the dashboard.

composer require ahsan-ul-alam/laravel-bisbond
php artisan bisbond:install
http://your-app.test/bisbond

Dashboard

The dashboard is the main setup center. From here, developers and users can:

  • See business setup status
  • Check invoice, SMS, and payment readiness
  • Enable or disable modules
  • Open quick actions to configure each section
  • Inspect routes and commands
  • Read developer help and integration guidance

Bangla Formatter

Use built-in helpers to format digits, money, and dates.

bn_digits(1234);   // ১২৩৪
bn_money(1500);    // ৳১,৫০০.০০
bn_date(now());    // Bangla formatted date

Invoice Module

Invoice preview is generated from configured business and invoice settings.

  • Business name
  • Invoice prefix
  • Footer note
  • Phone and address
  • Bangla formatted amount

Future expansion: PDF export and multiple invoice templates.

SMS Module

Configure your SMS provider and maintain reusable templates from the dashboard.

Templates:
- OTP
- Order Confirmation
- Payment Success

Placeholders:
{name}, {otp}, {amount}, {invoice_no}

Payment Architecture

Laravel Bisbond is designed to support local payment integrations through provider-based architecture.

bKash Nagad SSLCommerz Rocket Manual Payment

Future-ready: provider interface, registry, and dashboard-based provider configuration.

Developer Guide

Use this package as a dashboard-first toolkit for Laravel regional business apps.

  • Route prefix: /bisbond
  • Settings are stored in bisbond_settings
  • Use helpers to fetch settings and module status
  • Use provider-based architecture for future SMS and payment integrations
  • Publish config and views when customization is needed
